# Python en detalle

## sintaxis-y-tipos-en-python

Python se distingue por una sintaxis limpia que usa la indentación significativa para delimitar bloques de código, en lugar de llaves o palabras clave de cierre. Una sentencia `if`, un bucle `for` o una definición de función abren un bloque tras los dos puntos, y todo lo indentado bajo ellos les pertenece. Esta decisión, ideada por Guido van Rossum, obliga a un formato visualmente coherente. Las sentencias no requieren punto y coma final, y los comentarios comienzan con almohadilla.

Python es de tipado dinámico —las variables no declaran su tipo, que se infiere en ejecución— y de tipado fuerte, pues no realiza conversiones implícitas peligrosas entre tipos incompatibles. Los tipos numéricos incluyen `int`, de precisión arbitraria (puede representar enteros enormes sin desbordarse), `float` de doble precisión y `complex` para números complejos. El tipo `str` representa cadenas de texto inmutables Unicode, y `bool` los valores `True` y `False`. El valor especial `None` indica ausencia de valor. La asignación crea referencias a objetos; todo en Python es un objeto, incluso los números y las funciones. Desde Python 3.5 existen las anotaciones de tipo (type hints), opcionales y no obligatorias en ejecución, que permiten documentar tipos esperados y habilitar verificadores estáticos como mypy. La función `type()` revela el tipo de un objeto y `isinstance()` comprueba pertenencia, útiles para introspección.

## estructuras-de-datos-python-listas-dict-set

Python ofrece estructuras de datos integradas potentes y expresivas. La lista (`list`) es una secuencia ordenada y mutable que admite elementos de cualquier tipo, se define con corchetes y permite indexación, rebanado (slicing), inserción y eliminación; internamente es un arreglo dinámico. La tupla (`tuple`), definida con paréntesis, es similar pero inmutable, lo que la hace adecuada para datos que no deben cambiar y permite usarla como clave de diccionario. El rebanado, con la sintaxis `secuencia[inicio:fin:paso]`, es una de las características más elegantes para extraer subsecuencias.

El diccionario (`dict`) es una tabla hash que asocia claves únicas con valores, definido con llaves y pares clave-valor. Ofrece acceso, inserción y borrado en tiempo promedio O(1), y desde Python 3.7 conserva el orden de inserción. Es la estructura de datos más versátil del lenguaje, omnipresente en el manejo de datos estructurados, configuraciones y conteos. El conjunto (`set`) almacena elementos únicos sin orden, también basado en hashing, ideal para eliminar duplicados y realizar operaciones matemáticas de conjuntos como unión, intersección y diferencia con operadores intuitivos. Existen variantes especializadas en el módulo `collections`, como `defaultdict`, `Counter`, `deque` y `namedtuple`, que cubren necesidades frecuentes. Elegir la estructura adecuada según el patrón de acceso —orden, mutabilidad, unicidad, búsqueda por clave— es clave para escribir código eficiente y claro.

## funciones-comprehensions-y-generadores

Las funciones en Python se definen con la palabra clave `def`, reciben parámetros y pueden devolver valores con `return`. Soportan argumentos posicionales, con nombre, valores por defecto, y los flexibles `*args` (tupla de posicionales variables) y `**kwargs` (diccionario de nombrados variables). Las funciones son objetos de primera clase: pueden asignarse, pasarse y retornarse, habilitando programación funcional con `map`, `filter`, `lambda` (funciones anónimas) y los decoradores, que envuelven funciones para añadir comportamiento como registro, caché o control de acceso de forma elegante.

Las comprensiones (comprehensions) son una sintaxis concisa y pythónica para construir colecciones a partir de iterables. Una comprensión de lista como `[x*x for x in rango if x % 2 == 0]` genera una lista aplicando una expresión y un filtro opcional en una sola línea legible. Existen también comprensiones de diccionario y de conjunto. Los generadores son funciones que usan `yield` en lugar de `return` para producir valores perezosamente, uno a la vez, sin construir toda la secuencia en memoria; son ideales para procesar grandes volúmenes de datos o flujos potencialmente infinitos con eficiencia de memoria. Las expresiones generadoras, con paréntesis en lugar de corchetes, ofrecen la misma pereza de forma compacta. Los iteradores, que implementan el protocolo `__iter__` y `__next__`, sustentan los bucles `for`. Estas herramientas reflejan la influencia funcional en Python y producen código expresivo y eficiente.

## clases-y-modulos-en-python

Python soporta plenamente la programación orientada a objetos. Una clase se define con `class` y agrupa atributos y métodos. El método especial `__init__` es el constructor que inicializa cada instancia, y el primer parámetro de los métodos, por convención llamado `self`, referencia a la propia instancia. Python emplea numerosos métodos especiales o "dunder" (double underscore) como `__str__` para la representación en texto, `__len__` para la longitud, `__eq__` para la igualdad y `__repr__` para la depuración, que permiten que los objetos se integren con la sintaxis del lenguaje (sobrecarga de operadores).

La herencia se expresa indicando la clase base entre paréntesis, y Python admite herencia múltiple, resuelta mediante el orden de resolución de métodos (MRO) con el algoritmo C3. El encapsulamiento es por convención: un guion bajo inicial sugiere uso interno, y un doble guion bajo activa el "name mangling". Las propiedades con el decorador `@property` permiten controlar el acceso a atributos como si fueran campos. Los módulos organizan el código: cada archivo `.py` es un módulo importable con `import`, y los paquetes son carpetas con módulos relacionados. La biblioteca estándar de Python es célebre por su amplitud, con módulos para fechas, expresiones regulares, JSON, sistemas de archivos y mucho más, reflejando la filosofía de "baterías incluidas". El bloque `if __name__ == "__main__"` distingue cuándo un archivo se ejecuta directamente o se importa.

## manejo-de-archivos-y-excepciones

El manejo de archivos en Python es directo y seguro gracias a los gestores de contexto. La función `open()` abre un archivo en modos como lectura ('r'), escritura ('w'), adición ('a') o binario ('b'), devolviendo un objeto archivo. La práctica recomendada es usar la sentencia `with open(...) as f:`, que garantiza el cierre automático del archivo al salir del bloque, incluso si ocurre un error, evitando fugas de recursos. Dentro del bloque se puede leer con `read()`, `readline()` o iterando línea por línea, y escribir con `write()`. Para datos estructurados, los módulos `csv` y `json` facilitan la serialización y deserialización, y `pathlib` ofrece una API moderna y orientada a objetos para rutas del sistema de archivos.

El manejo de excepciones permite responder a errores en tiempo de ejecución sin que el programa colapse. El bloque `try` contiene el código que puede fallar, `except` captura excepciones específicas como `ValueError`, `FileNotFoundError` o `KeyError`, `else` se ejecuta si no hubo error, y `finally` siempre se ejecuta, ideal para limpieza. Es buena práctica capturar excepciones concretas en lugar de la genérica `Exception`, para no ocultar fallos inesperados. Se pueden lanzar excepciones con `raise` y definir clases de excepción propias heredando de `Exception`, modelando errores del dominio. Las excepciones forman una jerarquía de clases, lo que permite capturar familias enteras de errores. Este modelo de "es más fácil pedir perdón que permiso" (EAFP) es idiomático en Python, prefiriendo intentar la operación y manejar el fallo a verificar condiciones de antemano.

Conexiones: Python como sistema formal hereda su semántica de la lógica y la teoría de lenguajes; sus estructuras de datos (diccionarios como tablas hash, conjuntos con operaciones de unión e intersección) encarnan conceptos de matemáticas y teoría de conjuntos; las comprensiones y generadores reflejan ideas del cálculo lambda y la evaluación perezosa de la programación funcional; y el diseño legible del lenguaje conecta con la epistemología y la comunicación clara del conocimiento.
